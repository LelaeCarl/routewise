About RouteWise

A graduation-project web system for transparent, academically-aligned decision-making in cross-border logistics between China and Kenya.

Design goal

Project purpose

Provide a practical decision support tool that converts route-planning trade-offs (cost, time, and transport modes) into a clear interface operators can use — and researchers can evaluate.

Problem being solved

Multi-modal complexity

Sea, air, rail, and road handoffs across hubs/ICDs.

Cost vs time

Different lanes produce different delivery profiles.

Non-intuitive planning

Operators need clarity, not raw graph outputs.

Decision support gap

Small traders benefit from transparent tools.

Key features

  • Route planning with direction, endpoints, weight, and preference inputs.
  • Multi-modal routing across ports, airports, ICDs, and road handoffs.
  • Optimization modes (cheapest, fastest, balanced) to support clear comparisons.
  • Results with structure — KPIs, timeline legs, and comparison across objectives.

Technical approach

RouteWise is built as a Flask + Jinja web application with Dijkstra-based routing and possible A* extension.

Stack

Backend Python + Flask
UI Jinja + Tailwind CDN
Client JavaScript (lightweight)

Academic value

The UI is intentionally structured so future work can expose assumptions behind cost/time computations. This helps students and evaluators validate correctness and interpret trade-offs.
